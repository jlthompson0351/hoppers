QUICK DEPLOYMENT GUIDE - Copy to Pi
===================================

NOTE:
- This is a legacy human/manual copy guide.
- For Cursor/OpenClaw agent work, prefer Desktop Commander-launched remote commands instead of raw shell SSH/SCP.
- Copying files only stages them on the Pi; the update is not live until the service is restarted in an approved window and validated.

This folder contains deploy scripts.
Copy the **live source files** from `src/` (not the legacy duplicates in this folder).

METHOD 1 - Using WinSCP (EASIEST):
----------------------------------
1. Open WinSCP
2. Connect to: 172.16.190.25 (user: pi)
3. Navigate to: /opt/loadcell-transmitter/
4. Copy these files:

   FROM project source               TO Pi location:
   -------------------               ---------------
   ../src/app/routes.py                     -->   /opt/loadcell-transmitter/src/app/routes.py
   ../src/services/acquisition.py           -->   /opt/loadcell-transmitter/src/services/acquisition.py
   ../src/db/repo.py                        -->   /opt/loadcell-transmitter/src/db/repo.py
   ../src/core/zero_tracking.py             -->   /opt/loadcell-transmitter/src/core/zero_tracking.py
   ../src/core/throughput_cycle.py          -->   /opt/loadcell-transmitter/src/core/throughput_cycle.py
   ../src/core/post_dump_rezero.py          -->   /opt/loadcell-transmitter/src/core/post_dump_rezero.py
   ../src/core/zeroing.py                   -->   /opt/loadcell-transmitter/src/core/zeroing.py
   ../src/app/templates/dashboard.html      -->   /opt/loadcell-transmitter/src/app/templates/dashboard.html
   ../src/app/templates/hdmi.html           -->   /opt/loadcell-transmitter/src/app/templates/hdmi.html
   ../src/app/templates/settings.html       -->   /opt/loadcell-transmitter/src/app/templates/settings.html
   ../src/app/templates/scale_settings.html -->   /opt/loadcell-transmitter/src/app/templates/scale_settings.html
   ../src/app/templates/calibration.html    -->   /opt/loadcell-transmitter/src/app/templates/calibration.html

5. After copying, open PuTTY or SSH and run:
   ssh pi@172.16.190.25
   sudo systemctl restart loadcell-transmitter

6. Check it worked:
   sudo systemctl status loadcell-transmitter

7. Post-deploy for v3.4 (Job Target Mode):
   - Go to Settings > Job Target Mode tab
   - Set a webhook token in Webhook Security
   - Trigger signal dropdown will populate from existing PLC profile points
   - Toggle "JOB TARGET MODE" on dashboard when ready to test

AFTER DEPLOYMENT:
----------------
1. Go to http://172.16.190.25:8080
2. Press Ctrl+Shift+R to hard refresh
3. Confirm Settings page shows "Post-Dump Re-Zero (One-Shot)" section
4. If hopper is empty/stable: press ZERO once (manual confirm)
5. Enable/verify AZT + Post-Dump Re-Zero settings


WHAT THIS UPDATE DOES (v3.4):
-----------------------------
✓ Job Target Signal Mode: webhook-driven PLC output. Scale tells PLC when to stop filling.
  - POST /api/job/webhook receives target weight from external system
  - Output = 0V while below target, trigger voltage when at/above target
  - Trigger signal selectable from PLC profile dropdown in Settings
  - Pretrigger offset for falling-weight compensation
  - Dashboard mode toggle (Legacy / Job Target Mode) + job status bar
  - Dedicated Settings tab for target mode configuration
  - Token auth on all webhook endpoints (X-Scale-Token header)
✓ Previous: Industrial two-layer auto-zero, post-dump re-zero, throughput spike guard
